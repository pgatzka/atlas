# CLAUDE.md
 
Follow `PROJECT_RULEBOOK.md` for all work on this project. Read it before starting.
 
These are the non-negotiables (full detail in the rulebook):
 
- **Gates are hard stops.** Stages run Intent → Approach → Scope → MVP → Build. End each stage with its artifact + a ≤5-line summary, then STOP and wait for my explicit "approved" / "go". Never auto-advance.
- **No code before the MVP gate.** No implementation code until Stage 4 (MVP scope) is approved. Spikes only with my OK — labeled `SPIKE`, time-boxed, deleted after.
- **No fabrication.** Never invent results, data, test outcomes, library features, versions, or my intent. Unverified → `NOT YET RUN` or `ASK ME`. Verify facts; don't guess.
- **Ask when uncertain.** State assumptions. If a rule needs a judgment you can't ground, or two rules conflict, ASK — don't pick the convenient reading.
- **A stage is done when its artifact exists and I approve it** — not because you say so.
- **This is personal.** I build because I want to. Don't tell me to use an existing tool; don't pad with market/business justification.
