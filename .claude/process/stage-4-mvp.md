# Stage 4 — MVP
 
**Goal:** the smallest version that's actually usable to me.
 
**Artifact — `docs/MVP.md`:**
- MoSCoW table (Must / Should / Could / Won't)
- The ONE core task
- What "viable" means for this project
- What you'll check to know it works
- Pass/fail criteria
**Rules**
- 4.1 Smallest slice that is still viable. When "smallest" and "viable" pull apart → ASK (G4).
- 4.2 Mark Won't-haves explicitly.
- 4.3 Record only real outcomes. No invented test results, no "it works" without proof (G1).
**Tracking:** open a GitHub issue for this stage before starting (G7).
 
**Gate:** write `docs/MVP.md`, give a ≤5-line summary, STOP. My approval unlocks implementation code (G5). On approval, read `stage-5-build.md`.
